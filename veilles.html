🎯 Introduction & Problématique

Ma veille technologique porte sur l'évolution du langage Python et son rôle dans l'informatique moderne.

Aujourd'hui, Python est utilisé partout : web, data, intelligence artificielle et automatisation. Mais il souffre d'une limite historique : des performances inférieures aux langages compilés comme C++ ou Rust.

👉 Problématique : Comment Python reste-t-il le langage le plus utilisé tout en améliorant ses performances ?

Ma veille montre que Python évolue grâce à de nouvelles versions majeures et à l'intégration croissante de Rust dans son écosystème. Je m'appuie sur des flux RSS (Feedly), des newsletters tech et des sites spécialisés pour suivre cette actualité.

📅 PHASE 1
Jan. 2025
Python 3.13 : Plus de puissance
Libération du GIL · Compilateur JIT
▼

Avec Python 3.13, le langage franchit une étape majeure. La grande nouveauté est la suppression optionnelle du GIL (Global Interpreter Lock), un verrou qui empêchait l'exécution simultanée de plusieurs threads Python. Sa levée permet d'exploiter pleinement les processeurs multi-cœurs modernes.

⚡ Python gagne en vitesse grâce au multithreading réel et à un début de compilateur JIT qui traduit le code à la volée pour l'exécuter plus rapidement.
🔗 Le Monde Informatique 📄 Documentation officielle
Fév. 2025
Polars : La rapidité de Rust en Data Science
Data Science · Multithread · Alternative Pandas
▼

J'ai découvert Polars, une librairie de traitement de données pour Python, entièrement développée en Rust. Contrairement à Pandas, qui est limité par le GIL et fonctionne sur un seul cœur, Polars exploite le multithreading natif de Rust pour paralléliser les calculs automatiquement.

📊 Résultat concret : traitement de datasets jusqu'à 10 à 30 fois plus rapide que Pandas sur de gros volumes de données, avec une syntaxe proche et une API moderne.
🔗 Journal du Net 📄 Site officiel Polars
📅 PHASE 2
Mai 2025
uv : La gestion de projet Python réinventée
Outillage · Environnements · Rapidité
▼

L'outil uv, développé par Astral (les mêmes que Ruff), simplifie radicalement la gestion des projets Python. Il remplace en un seul binaire : pip, venv, pyenv et pip-tools. Écrit en Rust, il s'impose comme le remplaçant moderne de la chaîne d'outils classique.

🚀 Création d'un environnement virtuel et installation des dépendances en moins d'1 seconde — contre 30 secondes ou plus avec pip classique. Un gain de temps considérable au quotidien.
🔗 IT for Business 📄 GitHub astral-sh/uv
Sept. 2025
L'essor de l'IA locale avec Ollama
Confidentialité · Performance · LLM on-premise
▼

Une tendance forte émerge : l'IA locale. Avec des outils comme Ollama ou LM Studio, il est désormais possible de faire tourner des modèles de langage (LLM) directement sur son PC, sans envoyer de données à des services cloud externes. Python est le langage central pour interagir avec ces modèles via des API locales.

🔒 Avantages clés : confidentialité totale des données (enjeu critique en entreprise), zéro coût d'API et possibilité de travailler hors ligne. Un atout direct pour la cybersécurité.
🔗 ZDNet France 📄 ollama.com
Jan. 2026
L'ère des Agents IA autonomes
Automatisation · Supervision · DevOps
▼

Les agents IA (comme GitHub Copilot Agent, Devin ou Claude Code) sont capables de travailler de façon autonome : écrire du code, corriger des bugs, lancer des tests et créer des pull requests sans intervention humaine. Python est le langage le plus utilisé pour orchestrer ces agents via des frameworks comme LangChain ou CrewAI.

🤖 Le rôle du développeur évolue : on passe de l'écriture de code ligne à ligne à la supervision et prompt-engineering d'agents IA. Une compétence clé pour 2026 et au-delà.

🔬 Zoom Technique : Python × Rust

Pourquoi Rust est devenu le moteur caché de l'écosystème Python ?

Paradoxalement, beaucoup des outils Python les plus modernes sont écrits en Rust pour bénéficier de ses performances et de sa sécurité mémoire. Python garde sa facilité d'utilisation, tandis que Rust assure la vitesse sous le capot.

⚡ Ruff Linter

Linter et formateur Python. Remplace flake8, isort et black. 100× plus rapide.

🛡️ Pydantic V2 Validation

Validation de données 20× plus rapide grâce à un cœur Rust. Utilisé dans FastAPI.

📦 Polars Data

Alternative Rust à Pandas. Multithreading natif, 10-30× plus rapide sur gros datasets.

🚀 uv Outillage

Gestionnaire de projet Python. Remplace pip + venv + pyenv en un seul outil ultra-rapide.

💡 Analyse : Python conserve sa simplicité et sa popularité, mais gagne en puissance grâce à Rust. Cette synergie est centrale dans l'informatique moderne en 2026 — que ce soit en data science, IA ou cybersécurité.
📌 Synthèse globale — Axe Python & Rust
Conclusion de la veille

Ma veille confirme que Python reste le langage dominant grâce à une évolution constante.

Il gagne en performance avec Python 3.13 (suppression du GIL, JIT) et grâce à l'adoption massive de Rust dans son écosystème d'outils.

Il se renforce dans l'IA, la data et l'automatisation — des domaines en forte croissance dans les entreprises.

👉 Conclusion : Python évolue pour rester dominant, et Rust en est le catalyseur invisible.

🎯 Introduction & Problématique

Cette seconde veille porte sur la cybersécurité des applications web, un domaine directement lié à mon projet d'alternance en sécurité informatique.

En BTS SIO SLAM, j'ai développé plusieurs applications web (PHP, Symfony, MySQL). Mais concevoir une appli ne suffit pas : encore faut-il qu'elle soit sécurisée. Les failles applicatives représentent aujourd'hui la première cause de compromission des systèmes d'information.

👉 Problématique : Comment sécuriser une application web contre les menaces actuelles les plus courantes ?

Je me base sur le référentiel OWASP Top 10, des articles spécialisés (ANSSI, CERT-FR, ZDNet Sécu), et ma pratique sur la plateforme TryHackMe.

🔴 PHASE 1 — Les menaces
Sept. 2025
OWASP Top 10 : Les failles web incontournables
Référentiel · Injection · Authentification
▼
🔴 Impact critique

L'OWASP Top 10 est le référentiel mondial des risques de sécurité web. Il classe les 10 vulnérabilités les plus répandues et les plus dangereuses pour les applications. En tant que développeur SLAM, connaître ces failles est indispensable pour écrire du code sécurisé dès la conception (approche Security by Design).

  • A01
    Broken Access Control
    Accès non autorisé à des ressources ou fonctions protégées.
  • A02
    Cryptographic Failures
    Données sensibles mal chiffrées ou exposées en clair.
  • A03
    Injection (SQL, XSS…)
    Injection de code malveillant via des entrées non filtrées.
  • A07
    Identification & Auth Failures
    Mots de passe faibles, sessions non sécurisées, absence de 2FA.
🛡️ Dans mes propres projets Symfony et PHP, j'ai appliqué des requêtes préparées contre les injections SQL, le hachage bcrypt pour les mots de passe, et la validation des entrées côté serveur.
🔗 OWASP Top 10 📄 CERT-FR
Oct. 2025
Injections SQL & XSS : Comment s'en protéger
Injection · Cross-Site Scripting · PHP / Symfony
▼
🔴 Impact critique

L'injection SQL consiste à insérer du code SQL malveillant dans un formulaire pour manipuler la base de données — lire des données confidentielles, supprimer des tables ou contourner une authentification. Le XSS (Cross-Site Scripting) permet d'injecter du JavaScript dans une page vue par d'autres utilisateurs, pour voler leurs sessions ou les rediriger.

💉 Contre-mesures : requêtes préparées PDO (PHP) ou Doctrine ORM (Symfony) contre l'injection SQL, échappement des sorties avec htmlspecialchars() ou Twig contre le XSS, et Content Security Policy dans les headers HTTP.
🔗 Guide ANSSI 📄 PortSwigger Labs
🟣 PHASE 2 — Les bonnes pratiques
Nov. 2025
Authentification sécurisée & JWT
Sessions · Tokens · 2FA · Hachage
▼
🟠 Impact important

La gestion de l'authentification est l'un des points les plus critiques d'une application. Un mot de passe stocké en clair, une session sans expiration ou un token JWT sans signature correcte peuvent compromettre l'ensemble d'un système. Les bonnes pratiques modernes incluent le hachage bcrypt/Argon2, les tokens JWT signés pour les API, et la double authentification (2FA).

🔑 Dans le projet Campus HelpDesk, j'ai implémenté des sessions PHP sécurisées avec régénération d'ID de session après connexion et des mots de passe hachés en bcrypt — mesures directement issues de cette veille.
🔗 OWASP Auth Cheat Sheet
Jan. 2026
HTTPS, Headers HTTP & RGPD
Chiffrement · Conformité · CSP · HSTS
▼
🟠 Impact important

Le HTTPS (TLS/SSL) est aujourd'hui obligatoire pour chiffrer les échanges entre le navigateur et le serveur. Mais la sécurité HTTP ne s'arrête pas là : des headers comme Content-Security-Policy (CSP), HSTS, X-Frame-Options ou Referrer-Policy renforcent considérablement la protection contre les attaques XSS, clickjacking et MITM. Le RGPD impose par ailleurs des obligations légales sur la collecte et le traitement des données personnelles.

🌐 Un outil comme Security Headers (securityheaders.com) permet d'analyser les headers d'un site en quelques secondes et d'identifier les manques — une pratique que j'applique sur mes propres projets.
🔗 Security Headers 📄 CNIL — RGPD

🔬 Zoom Technique : Sécurité applicative en pratique

Les outils du quotidien pour auditer et renforcer une application web.

Plusieurs outils permettent d'identifier rapidement les vulnérabilités d'une application web, que ce soit en phase de développement ou lors d'un audit de sécurité.

🕷️ Burp Suite Pentest

Proxy d'interception HTTP. Permet d'analyser et modifier les requêtes en temps réel pour tester les failles.

🔍 OWASP ZAP Scanner

Scanner de vulnérabilités web open-source. Alternative gratuite à Burp Suite, idéal pour débuter.

🌐 Nmap Réseau

Scanner de ports et de services. Permet de cartographier une infrastructure et d'identifier les services exposés.

🔐 Kali Linux OS dédié

Distribution Linux spécialisée en pentest. Inclut nativement Burp Suite, Nmap, Wireshark et +600 outils sécu.

💡 Analyse : Mon background SLAM (développement PHP/Symfony/MySQL) est un atout direct en cybersécurité applicative : je comprends comment les failles naissent dans le code, ce qui me permet de mieux les identifier et les corriger. C'est la valeur ajoutée du profil "dev qui comprend la sécu".

Conclusion de la veille

Cette veille m'a permis de comprendre que la sécurité n'est pas une étape finale mais une démarche intégrée dès le développement (Security by Design).

Les failles comme l'injection SQL, le XSS ou les défauts d'authentification sont évitables si le développeur applique les bonnes pratiques OWASP dès l'écriture du code.

Mon profil SLAM me permet d'aborder la cybersécurité avec une vision concrète : je sais comment les vulnérabilités naissent dans le code, et comment les corriger.

👉 Conclusion : Développeur et sécurité ne s'opposent pas — ils se complètent. Mon objectif en alternance est d'appliquer cette double compétence.
